Remote for Claude

사용자 PC의 Claude Code를 브라우저/모바일에서 다루는 제어면입니다. 사이트는 identity, pairing, relay, UX를 맡고, 실제 실행은 PC connector가 맡습니다.

역할

제품 아키텍처, connector runtime, backend relay, frontend UX, 디바이스 생명주기, 릴리즈 패키징, 문서화를 설계·구현했습니다.

문제

Claude Code는 사용자의 repo, shell, token이 있는 PC에서 가장 강합니다. 하지만 그만큼 작업 흐름이 한 키보드에 묶입니다.

접근

실행은 로컬에 둡니다. 페어링된 connector daemon 앞에 좁은 브라우저/모바일 제어면을 두고, 로컬 behavior를 구동하는 데 필요한 것만 relay합니다.

상태

Alpha 제품. SaaS/self-host wiring, multi-device pairing, remote chat, diagnostics, unpair cleanup, 테스트, 빌드, 다국어, 릴리즈 패키징이 연결되어 있습니다.

§ 01 · Architecture

원격 UI, 로컬 실행

사이트는 제품처럼 느껴져야 하지만, 민감한 실행 표면은 사용자의 PC에 남아야 합니다.

End-to-end path
Browser → Worker → DO → Connector → Claude

인증과 라우팅은 edge에서, 작업과 파일은 페어링된 PC에.

SiteSolidJS, i18n, quota, device settings
WorkerHono routes, D1 devices, KV replay guard
DO RelayOutbound WebSocket, request caps, stream relay
PCBun daemon, local Bearer auth, remote-claude behavior
hub바깥은 제품 shell, 안쪽은 사용자 PC의 runtime authority.
System slices

Frontend. SolidJS + Vite SPA. 4개 locale, device picker, chat, diagnostics, approval modal, workspace picker, Android Capacitor wrapper.

Backend. Cloudflare Workers + Hono. D1은 user/device/pairing/license 상태를 저장하고, KV는 replay guard, Durable Object는 per-device outbound WebSocket relay 상태를 잡습니다.

Connector. 데스크톱 배포용 Bun daemon. 로컬에 device identity를 저장하고, 인증된 local HTTP surface를 열고, site relay로 dial out하고, behavior를 host합니다.

Behavior runtime. remote-claude behavior가 Claude Code 세션 시작/재개, 이벤트 스트리밍, 승인 중재, workspace guard를 담당합니다.

§ 02 · Device lifecycle

페어링과 해제는 운영 기능이 아니라 제품 UX입니다

어려운 부분은 디바이스를 추가하는 것만이 아닙니다. 디바이스가 사라질 때 브라우저, 백엔드, relay, 로컬 connector 상태를 모두 정직하게 정리하는 것입니다.

Pair

일회용 코드와 로컬 identity

사이트가 짧게 살아있는 코드를 발급합니다. connector가 pairing을 완료하고 Ed25519 기반 identity와 connection token을 저장한 뒤 relay에 붙습니다.

Operate

진단과 workspace guard

connector 상태, 로드된 behavior, workspace roots, approval hook, Claude CLI 버전 등을 device settings에서 확인할 수 있습니다.

Unpair

live relay를 끊고 stale state를 지움

삭제 시 selected device 상태, per-device browser preference, Durable Object relay 상태를 정리합니다. 로컬 connector identity는 명시적 CLI unpair 명령으로 지웁니다.

§ 03 · Trust boundary

현재는 thin storage, 다음은 더 강한 transport

Alpha 구현은 채팅 transcript와 사용자 코드를 D1, KV, 로그에 저장하지 않습니다. hosted-code assistant보다 얇은 구조입니다. 동시에 release boundary도 명시했습니다. 현재 outbound WebSocket relay는 사이트 프로세스를 통해 평문 request body가 지나가고, WebRTC/E2E body encryption은 다음 trust-model hardening 트랙입니다.

이 구분은 중요합니다. 아직 아키텍처가 강제하지 않는 privacy claim을 앞세우기보다, 코드와 문서가 같은 경계를 말하게 만드는 쪽을 선택했습니다.
