커널

Ops-Cure — 채널 단위 state/event 커널

채널은 단순한 메시지 파이프가 아닙니다. 액터와 이벤트가 흐르는 상태 공간입니다. Ops-Cure는 그 해석을 명시화해서, behavior는 꽂히고 transport는 맞춰지고 runtime은 실행합니다.

Channel-native kernel
Space · Actor · Event · Behavior

커널은 범용으로 유지합니다. 도메인 시맨틱은 behavior 안에, 머신 별 실행 세부는 runtime 안에.

Core vocabulary

Space. 상태를 가진 방 — 시간 축으로 자체 상태를 누적하는 Discord 채널, 스레드, 세션.

Actor. 참여자 — 사람, AI 워커, 봇, 시스템.

Event. 공간 안에 기록되는 사실. Transport 독립적, 저장/재생 가능.

Behavior. 커널 위에서 이벤트를 해석하고 도메인 상태를 관리하는 도메인 로직.

하나의 커널, 세 개의 behavior

같은 Space/Actor/Event 기반 위에서 프로덕션급 멀티에이전트 워크플로, 브라우저 기반 원격 실행 면, Codex끼리 나누는 대화방이 돌아갑니다. 서로 완전히 다른 도메인이 같은 커널을 쓴다는 것 — 추상화가 실제로 성립한다는 증거입니다.

왜 세 개가 중요한가

behavior 하나는 기능입니다. 근본적으로 다른 behavior 세 개 — 구조화된 워크플로, 승인이 붙은 원격 실행, 자유 대화 — 가 같은 커널 위에서 돌아간다는 건 추상화가 실체라는 주장입니다. 각각이 다른 축을 검증합니다: orchestration 은 상태/태스크 모델, remote_codex 는 분산 runtime 아래의 canonical evidence, chat 은 태스크 시맨틱 없는 방의 범용성.

Remote for Claude — 내 PC의 Claude Code를 브라우저/모바일에서 제어

사용자가 여러 PC를 페어링하고, 브라우저 또는 Android 앱에서 디바이스를 고른 뒤, 로컬 claude CLI를 실행하게 하는 SaaS 플랫폼입니다. 내부 behavior였던 원격 실행 아이디어를 실제 제품 구조로 옮긴 케이스입니다.

Browser/mobile → PC connector
Site · Relay · Connector · Behavior

사이트는 인증, 페어링, 디바이스 선택, relay를 담당합니다. 실행, 파일 접근, hook, behavior runtime은 사용자의 PC에 남깁니다.

FrontendSolidJS SPA, i18n, Android wrapper
BackendHono Workers, D1, KV, Durable Object relay
ConnectorBun daemon, Ed25519 identity, local auth
Behaviorremote-claude sessions, approvals, workspace guards
settings_remote원격 UI, 로컬 실행 경계.
구현 범위

제품 아키텍처. shared contract, behavior SDK, PC connector daemon/CLI, Cloudflare Workers backend, SolidJS frontend, Capacitor Android packaging까지 한 모노레포로 구성했습니다.

디바이스 생명주기. 페어링 코드, 디바이스 identity, 진단, 기기별 설정, unpair cleanup을 연결했습니다. 해제 시 live relay를 끊고 브라우저/디바이스 로컬 상태를 정리합니다.

런타임 경계. 사이트는 채팅 transcript나 코드를 저장하지 않습니다. alpha에서는 Durable Object를 통해 평문 relay가 지나가며, E2E/WebRTC hardening은 다음 trust-model 단계로 문서화했습니다.

릴리즈 표면. winget/Homebrew 템플릿, Android listing copy, localization source, operator runbook을 코드 옆에 둬서 데모가 아니라 배포 가능한 제품 흐름으로 정리했습니다.
