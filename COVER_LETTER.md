# Cover Letter

안녕하세요. 저는 Unity 기반 실시간 시스템을 중심으로 XR, 로보틱스, BIM 시각화, 에디터 툴링, 그리고 최근에는 AI 운영 구조까지 직접 만들고 다듬어온 홍대기입니다.

제가 가장 중요하게 보는 기준은 `실제로 운영 가능한가`입니다. 기능이 동작하는 것만으로는 충분하지 않고, 현장 설치와 검증, 운영자 안전, 반복 가능한 전달 방식까지 함께 버텨야 좋은 시스템이라고 생각합니다. 그래서 저는 구현만 하는 개발자보다, 구조를 먼저 보고 운영까지 닫는 엔지니어에 더 가깝습니다.

이 기준은 프로젝트 전반에 일관되게 이어졌습니다. VR Robot에서는 Quest 3 입력부터 Unity, Flask, RoboDK, 안전 로직, 자체 IK 확장까지 하나의 산업용 XR 제어 파이프라인으로 묶었습니다. WatchBIM에서는 대용량 BIM 데이터를 다루기 위한 히트 엔진, 단면 생성, 2D 분석 뷰어를 직접 만들며 geometry tooling까지 책임졌습니다. DXCenter에서는 비개발자도 사용할 수 있는 에디터 툴을 통해 반복 작업 비용을 줄이고 운영 구조를 안정화했습니다.

최근에는 `Ops-Cure`를 통해 AI 쪽에서도 같은 문제를 풀고 있습니다. Discord를 제어면으로, Windows 로컬 CLI를 실행면으로 분리하고, NAS bridge와 launcher, markdown workspace, worker heartbeat를 연결해 세션과 아티팩트가 눈에 보이는 로컬 에이전트 운영 구조를 만들었습니다. 저는 AI를 데모 기능으로 붙이기보다, 규칙과 상태 가시성, handoff, drift 감지까지 포함해 실제 운영 컴포넌트로 다루는 방식에 관심이 많습니다.

결국 제가 잘하는 일은 복잡한 시스템을 단순한 화면으로 포장하는 것이 아니라, 복잡한 시스템이 실제 환경에서 계속 굴러가게 만드는 것입니다. XR, 시뮬레이터, BIM, 로보틱스, AI 운영 구조처럼 제약이 많은 영역일수록 이런 방식이 더 중요하다고 믿습니다.

포트폴리오에는 대표 프로젝트와 작업 기준을 함께 정리해두었습니다. 더 자세한 맥락이나 특정 역할과의 연결점을 이야기해보고 싶다면 편하게 연락 부탁드립니다.
