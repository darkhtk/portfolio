Industrial systems

XR, robotics, and BIM runtime proof

Industrial XR x Robotics

VR Robot Teleoperation

A full pipeline from Quest 3 input to a Doosan A0912 robot through Unity, Flask, RoboDK, safety logic, calibration, and later an in-house Jacobian IK extension.
Problem: XR input needed to control an industrial robot with stable mapping, safe movement, and room for later technical ownership.
Build: Controller mapping, EMA filtering, coordinate transforms, bridge APIs, safety boxes, deadman switch logic, and in-house IK continuation.
Outcome: Public exhibition success at BEXCO and a strong proof of industrial XR that handled real control constraints.
BIM Runtime Tooling

WatchBIM

A Unity BIM runtime built around a custom hit engine, cross-section generation, 2D analysis viewer, and a tool-grade camera system for heavy geometry inspection.
Problem: Large BIM scenes needed accurate selection, stable camera behavior, and cross-section analysis inside one usable runtime tool.
Build: Triangle caching, hit filtering, geometry jobs, plane intersection, segment grouping, 3D to 2D conversion, and slope labeling.
Outcome: A practical inspection and analysis runtime that goes beyond visualization into geometry tooling ownership.
These two projects are the clearest proof that I can build runtime systems where control reliability and geometry complexity matter as much as the UI.

Tooling and interactive XR

Authoring systems and exhibition-grade AI XR

Simulation Tooling

DXCenter

A Unity authoring environment for digital-twin style workflows, built around nine EditorWindow tools and a repeatable content structure for non-developer teams.
Problem: Content teams were paying repeated cost whenever mapping, structure, and runtime checks had to be rebuilt by hand.
Build: EditorWindow stack, shared operating patterns, data mapping flow, site-level content structure, and one-click WebGL output.
Outcome: Lowered authoring friction and made content operations more consistent and reviewable.
Interactive XR x Generative AI

Neostalgia

An XR exhibition system that interviews visitors about memory, generates visual assets from the conversation, and keeps the whole experience operable through logging and a dashboard.
Problem: The experience had to coordinate voice input, AI generation, XR state transitions, and real exhibition operations.
Build: Memory interview orchestration, voice-to-AI pipeline, Skybox and Meshy asset generation, telemetry, and operations dashboard.
Outcome: BIFAN XR selection, Minister citation, 130+ direct participants, and an exhibition system that held up under live traffic.
