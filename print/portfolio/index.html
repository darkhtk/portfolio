Industrial systems

XR control and BIM runtime proof

Industrial XR x Robotics

VR Robot Teleoperation

A full pipeline from Quest 3 controller input to a Doosan A0912 robot through Unity, Flask, RoboDK, calibration, safety logic, and later in-house Jacobian IK ownership.

  • Problem: XR input needed to control an industrial robot with stable mapping, safe movement, and room for deeper technical ownership.
  • Build: Controller filtering, coordinate transforms, bridge APIs, safety boxes, deadman logic, resend control, and IK continuation.
  • Outcome: A working public demonstration and a clear proof of industrial XR that handled real control constraints.
BIM Runtime Tooling

WatchBIM

A Unity BIM runtime built around a custom hit engine, station-based cross-section generation, 2D analysis viewer, and a tool-grade camera system for heavy geometry inspection.

  • Problem: Large BIM scenes needed accurate selection, reliable movement, and in-runtime cross-section analysis.
  • Build: Triangle caching, hit filtering, geometry jobs, plane intersection, segment grouping, 3D to 2D conversion, and slope labeling.
  • Outcome: A practical viewer and analysis tool that goes beyond visualization into geometry and domain logic ownership.
These two projects are the strongest proof that I can build runtime systems where control reliability, geometry complexity, and operator usability matter equally.

Tooling and exhibition XR

Authoring systems and AI-connected XR experience

Simulation Tooling

DXCenter

A Unity authoring environment for digital-twin style workflows, built around nine EditorWindow tools and a repeatable content structure for non-developer teams.

  • Problem: Content mapping and runtime-safe authoring were too dependent on repeated developer intervention.
  • Build: EditorWindow stack, shared operating patterns, data mapping flow, site-level content structure, and cleaner WebGL output flow.
  • Outcome: Lowered authoring friction and made digital-twin content operations more stable and reviewable.
Interactive XR x Generative AI

Neostalgia

An XR exhibition system that interviews visitors about memory, generates visual assets from the conversation, and keeps the whole experience operable through telemetry and a dashboard.

  • Problem: Voice input, AI generation, XR sequencing, and exhibition operations had to stay coherent under live traffic.
  • Build: Memory interview orchestration, voice-to-AI pipeline, Skybox and Meshy generation, session logging, and operations dashboard.
  • Outcome: BIFAN XR selection, MSIT Minister Citation, 130+ direct participants, and an XR system that held up in public exhibition use.

Applied AI systems

Agent operations, orchestration, and local AI structure

Ops-Cure

Ops-Cure turns Discord into a control surface for local AI work. A NAS bridge keeps session state and artifacts visible, while a Windows launcher runs local workers behind fixed adapters.

  • Discord thread as session key.
  • FastAPI + SQLite control plane.
  • Windows launcher and YAML-driven worker presets.
  • Artifact heartbeat and drift-aware monitoring.
Discord thread

Planner status, workspace path, key files, and agent routing remain visible inside the session thread.

Artifact report

Bridge reports artifacts, stage status, and handoff-ready files without exposing raw shell access to operators.

Workflow framework

Multi-Agent Orchestration

Reusable role rules, review gates, launcher flow, and operating logic for agent collaboration.

Local LLM R&D

GenWorld + Ollama

NPC memory, relationship rules, quest hooks, and inspectable runtime structure for local LLM behavior.

Why this AI work matters
  • Moves beyond prompt experimentation into operating structure and platform thinking.
  • Connects AI to concrete outputs: worker routing, state visibility, XR timing, and gameplay data rules.
  • Fits teams building internal tools, agent infrastructure, and applied AI products with real operators.
The AI thread in this portfolio is practical by design: observability, routing, fallback structure, and runtime ownership matter more than flashy demos.
