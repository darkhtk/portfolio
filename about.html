문제를 보는 기준 업무를 풀어 가는 방식

저는 산업용 Unity 실시간 시스템을 오래 만들어 온 개발자입니다. XR, 시뮬레이션, BIM 시각화, 에디터 툴링, 자동화, 로봇 연동처럼 제약이 많은 문제를 실제 프로젝트에서 풀어 왔습니다.

최근에는 AI 협업을 함께 사용하고 있지만, 중심은 언제나 구조와 검증입니다.

저는 데모보다 운영 가능성, 빠른 시제품보다 유지보수 가능한 구조, 많은 기술보다 문제와 제약에 맞는 설계를 더 중요하게 봅니다.

How I Work

문제보다 먼저 제약을 봅니다

산업용 시스템에서는 성능, 안전, 운영 환경, 납품 방식, 수정 비용이 설계의 출발점입니다. 그래서 기능보다 제약을 먼저 봅니다.

반복 비용을 구조로 줄입니다

Editor Tooling, 자동화, 데이터 구조 설계, 패턴 분리로 반복 작업과 협업 병목을 줄이는 쪽에 자주 집중합니다.

AI는 속도를 높이되 통제는 유지합니다

AI는 구현 보조와 반복 축소에 쓰지만, 시스템 구조와 검증 기준은 사람이 잡아야 결과물이 오래 갑니다.

Where AI Fits

AI를 쓰는 이유

  • XR, Editor Tooling, 백엔드, 통신 모듈을 동시에 잡아야 할 때 반복 구현량이 많기 때문입니다.
  • 설계가 명확한 영역에서는 속도를 높이면서도 일관성을 유지할 수 있기 때문입니다.
  • 프로젝트마다 다시 처음부터 셋업하지 않도록 작업 기준을 반복 가능한 형태로 남겨 두기 위해서입니다.

AI를 쓰지 않는 방식

  • AI를 전면 성과처럼 내세우지 않습니다.
  • 도메인 제약이 불명확한 상태에서 결과를 그대로 믿지 않습니다.
  • 동작 검증, 안전 기준, 운영 판단을 AI에 맡기지 않습니다.
What I Have Solved

실시간 XR과 시뮬레이션

VR/AR 안전교육, 몰입형 시뮬레이터, 관객 반응형 XR 콘텐츠를 만들며 실제 상호작용 구조와 납품 완성도를 다뤘습니다.

BIM과 대용량 시각화

고속도로 BIM 3D 시각화, 설계 자동화, 횡단면 생성, 최적화 구조를 통해 대규모 데이터와 성능 문제를 풀었습니다.

Unity Editor Tooling과 자동화

콘텐츠 제작자와 운영 담당자가 직접 다룰 수 있는 에디터와 빌드 흐름을 만들며 협업 마찰을 줄였습니다.

로봇 연동과 실물 제어

VR 로봇 조종, 통신 브릿지, 안전 로직, 하드웨어 연동 콘텐츠를 통해 실시간 제어 계층을 직접 다뤘습니다.
