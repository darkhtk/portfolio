# CHANGELOG_PORTFOLIO_REWRITE

## 무엇을 바꿨는지

- 홈, Projects, About, Resume의 정보 구조를 다시 설계했다.
- 홈을 `Hero → How I Actually Use AI → Featured Projects → Foundation Projects → Working Style → Final CTA` 순서로 재배치했다.
- 홈과 Projects에서 대표 프로젝트를 3개만 전면에 두고, 각 프로젝트를 문제 해결형 구조로 다시 작성했다.
- 기존 산업 프로젝트들은 별도 Foundation 섹션으로 묶어 현재 AI 활용 방식의 기반 경력으로 재배치했다.
- About 페이지는 자기 브랜딩 문구보다 문제 해결 방식과 AI 사용 기준을 설명하는 페이지로 바꿨다.
- Resume 페이지는 단순 나열 대신 `교육용 게임 → VR/AR 안전교육 → BIM/설계 자동화 → Editor Tooling/디지털트윈 → AI 융합 XR/로봇 제어` 흐름이 보이도록 재정리했다.
- 핵심 페이지의 메타 타이틀과 설명도 같은 방향으로 다시 썼다.
- 시각 톤을 다크 브랜딩 중심에서 밝은 editorial 톤으로 전환했다.
- 추가로 `layout/` 기준을 반영해 홈과 Projects를 `Newsreader + Manrope`, 밝은 종이 톤, 배경 톤 전환 중심의 `Curated Ledger` 레이아웃으로 다듬었다.
- Featured Projects는 카드형 소개보다 문서형 dossier 구조에 가깝게 조정했고, Foundation Projects는 카드 그리드보다 아카이브 목록처럼 읽히도록 위계를 바꿨다.
- Cover Letter와 VR 로봇 case study의 전면 카피도 일부 객관화해 홈과 메시지 충돌이 덜 나도록 조정했다.

## 왜 그렇게 바꿨는지

- 기존 구조는 강점이 많았지만, 첫 인상이 "산업용 Unity 실시간 시스템 개발자"보다 "AI Agent 브랜딩이 강한 개발자"에 가깝게 읽혔다.
- 사용자 요청의 핵심은 AI 자체를 자랑하는 것이 아니라, 어려운 시스템을 구조적으로 풀어 실제 동작하는 결과물로 만든 사람처럼 읽히게 하는 것이었다.
- 그래서 대표 프로젝트 수를 줄이고, AI 사용을 별도 자랑 포인트가 아니라 문제 해결 과정의 한 단계로 설명하도록 바꿨다.
- 기존 프로젝트를 버리지 않고 foundation으로 재배치한 이유는, 현재의 AI 활용이 갑자기 생긴 것이 아니라 XR, BIM, 툴링, 자동화, 납품 경험 위에 올라온 것임을 보여 주기 위해서다.

## 메시지가 어떻게 더 명확해졌는지

- 홈 첫 인상에서 산업용 Unity 실시간 시스템 개발자라는 정체성이 먼저 읽히게 됐다.
- AI는 정체성의 중심 문구가 아니라, 최근 차별점을 설명하는 구조적 도구로 위치가 조정됐다.
- 대표 프로젝트 3개가 각각 다른 문제 축을 대표하게 되어, 사용자가 "이 사람이 어떤 문제를 풀 수 있는가"를 빠르게 파악할 수 있게 됐다.
- Foundation 섹션이 생기면서 과거 프로젝트가 산만한 목록이 아니라 현재 역량의 근거로 읽히게 됐다.
- About과 Resume도 홈의 반복이 아니라 각각 `일하는 방식`, `성장 흐름`이라는 역할이 분리됐다.
- 레이아웃 기준을 반영하면서 화면 전체가 "예쁜 포트폴리오"보다 "정리된 기록물"처럼 읽히게 됐다.
- 모바일에서도 정보량은 유지하되 카드 안의 읽기 순서를 고정해, 많이 한 사람보다 잘 정리된 사람처럼 보이도록 조정했다.

## 남은 개선 포인트

- 일부 case study는 아직 기존 다크 레이아웃 기반이어서, 필요하면 `Curated Ledger` 스타일로 전면 통일까지 한 번 더 진행할 수 있다.
- Projects 페이지에서 대표 3개 외 개별 상세 페이지를 더 늘릴지 여부는 향후 우선순위에 따라 결정 가능하다.
- 실제 브라우저 기반 시각 검수와 GitHub Pages 배포 후 최종 모바일 점검은 한 번 더 해보는 것이 좋다.

## 최종 정리 패스
- Hero 카피를 더 줄여서 AI보다 "산업용 Unity 실시간 시스템 개발자"가 먼저 읽히도록 다듬었다.
- AI 관련 설명은 일반적인 역량 강조보다 `문제 → 적용 → 검증 → 결과` 흐름이 먼저 보이도록 압축했다.
- case study 전면 라벨에서 `Self-Built`, `AI Agent`, `멀티 페르소나` 같은 표현을 줄이고 더 차분한 workflow 중심 문구로 바꿨다.
- Foundation과 보조 섹션의 텍스트 밀도를 낮춰, 많이 해 본 사람보다 잘 정리된 사람처럼 읽히도록 조정했다.
