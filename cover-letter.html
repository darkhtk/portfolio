홍 대기

산업용 Unity 실시간 시스템 개발자 · XR 3년 + 산업 시각화·자동화 4년 + AI 융합 2년

구조와 도구로 협업 비용을 줄이는 개발자

사람이 많아질수록 속도와 기준이 달라지고, 같은 목표를 보더라도 작업 방식은 쉽게 흔들립니다. 저는 이 마찰을 감정으로 풀기보다 구조와 도구로 줄이는 개발자입니다. 팀이 더 빠르게 같은 목표에 도달하도록 제작 흐름과 판단 기준을 정리하는 역할에 강점이 있습니다.

도구로 협업 비용을 줄이다

제가 조화를 만드는 방식은 반복 비용을 줄이는 구조를 먼저 만드는 것에서 시작합니다. 현재 와이엠엑스에서 진행한 두 가지 에디터 툴링 프로젝트가 그 예입니다. LG eGuide Editor에서는 매뉴얼 콘텐츠를 카테고리·순서도·스텝 단위로 구성하고, 다양한 미디어와 스텝 연동을 안정적으로 다룰 수 있게 설계했습니다. 그 결과 콘텐츠 제작자가 코드 수정 없이 교육 콘텐츠를 직접 생산할 수 있게 됐습니다. DXCenter에서는 건물→층→장비→파트 구조의 현장 관리 시스템을 9개 커스텀 EditorWindow로 정리하고, 상태 효과와 운영 흐름을 패턴 단위로 분리해 기획·제작·검수 전 과정의 마찰을 줄였습니다.

두 프로젝트 모두에서 QA 효율도 같은 관점으로 접근했습니다. 씬 컨텍스트에 맞는 UI만 노출하고, Play Mode 진입 시 편집을 자동 차단하며, Undo를 전 윈도우에서 지원하는 등 검수자가 실수하기 어려운 구조를 만들었습니다. 저는 개발 속도만큼이나 검증과 수정 속도를 높이는 것이 품질과 일정에 직결된다고 생각합니다.

DXCenter와 LG eGuide Editor는 제작자와 검수자가 같은 구조를 공유하도록 설계한 대표 사례입니다.
운영 규칙과 검수 흐름까지 같이 버전 관리한 AI 협업 실험 화면입니다.

대규모 데이터 처리와 성능 설계

장헌파트너스에서 4년 7개월간 고속도로 BIM 3D 시각화 플랫폼을 메인 개발했습니다. 수만 개에 달하는 BIM 모델을 Google Protocol Buffer 기반 DB로부터 Unity로 임포트할 때, 에디터 라이프사이클 안에서 멈추지 않고 빠르고 안정적으로 처리하는 설계가 필요했습니다. 저는 Vertices/Triangle 분석 기반으로 모델 타입별 LOD를 자동 생성하면서, 메시 생성 및 normal/UV 계산을 Unity Job System으로 병렬 처리해 병목을 제거했습니다. 그 결과 10,000개 모델 기준 3시간 걸리던 임포트를 20분으로 단축했고, Shader 기반 모델 타입별 시각 구분과 AABB/거리 기반 Clipping을 결합해 전체 고속도로 구간을 하나의 3D 환경에서 실시간으로 탐색할 수 있게 만들었습니다.

같은 프로젝트에서 다중 도로 횡단면 실시간 분석 알고리즘도 직접 설계·구현했습니다. 도로 선형(Alignment)의 StationPoint 기준으로 3D 메쉬를 평면으로 절단하고, 정점 분석으로 교차 라인을 추출하여 2D 횡단면을 자동 렌더링합니다. 절토(1:N)·성토(1:N)·노면(N%) 등 모델 타입별 경사도를 자동 계산하고, 스냅핑이 가능한 치수선 시스템과 모델 타입별 레이어 On/Off까지 구현했습니다. 도로 종단면도 자동 생성과 플레이어 위치·시야 동기화, Coroutine 기반 비동기 처리로 진행률 표시와 취소를 지원하며, 전체 공사 구역의 설계 검토 효율을 크게 향상시켰습니다.

BIM 모델 처리와 횡단면 자동 생성은 성능 최적화와 설계 검토를 함께 요구하던 핵심 작업이었습니다.
AI 적용 방식

AI를 어디에 적용했고, 어떤 방식으로 통제했는지

1

제품 파이프라인 적용

너스탤지아 XR 콘텐츠에서는 음성 입력, Whisper, GPT, TTS, Skybox AI, Meshy를 하나의 흐름으로 묶고, Webhook 비동기 처리와 사용자별 저장 구조, 운영 대시보드까지 연결했습니다.

2

개발 과정 보조

VR 로봇 원격 조종에서는 입력 안정화, Flask API, UDP 브릿지, 안전 로직 구현에 AI 협업을 보조적으로 사용했고, DXCenter에서는 9개 EditorWindow와 패턴 분리를 같은 기준으로 다뤘습니다.

같은 기준을 GenWorld에도 적용해 로컬 Ollama LLM 기반 NPC 인지·대화 구조를 검증했습니다.

3

운영 방식 추출

프로젝트마다 반복되던 컨벤션, 금지 패턴, 검증 흐름을 남겨 두고 다음 작업에서도 같은 기준을 유지하고 있습니다.

공학적 사고를 즐기는 개발자

저는 수학과 물리 기반으로 실제 환경을 상상하고 도식화하는 것을 좋아합니다. 해군 VR 시뮬레이터에서는 아지무스 추진기 조종 로직과 HMD+PC 멀티플레이를 설계했고, LG하우시스 VR 안전교육에서는 RS232 기반 4D 하드웨어 제어를 직접 구현했습니다. 건설현장 VR 시뮬레이터에서는 비계 구조와 물리 체험을 에셋 없이 구성했습니다. 최근에는 Jacobian 기반 IK 솔버를 직접 구현하며 좌표계 변환, 관절 제약, 수치 안정성, 특이점 회피를 다루고 있습니다. 이런 기반은 실시간 인터랙션, 물리, 카메라, 성능 최적화가 중요한 산업용 프로젝트에서 안정적인 밑바탕이 됩니다.

마무리

저는 제작 흐름과 운영 구조를 다듬고, 대규모 데이터 처리와 성능 문제를 설계로 해결하며, 최근에는 AI 협업을 XR·에디터·백엔드·로봇 제어 실무에 적용하고 있습니다. 중요한 것은 AI 자체보다 실제로 동작하는 시스템과 오래 유지할 수 있는 구조라고 생각합니다.
